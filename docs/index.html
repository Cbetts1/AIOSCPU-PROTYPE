// features

Everything in one boot

AIOSCPU Prototype One ships with a complete software-layer OS — kernel, CPU, filesystem, AI core, and a live terminal — all in a single Node.js process.

🧠

AIOSCPU v1.0 — Virtual Processor

Software-emulated 32-bit CPU with 30+ ISA opcodes across 6 instruction categories. 64 KB flat memory, 8 general-purpose registers, hardware stack, and a 10 M-cycle safety cap.

⚙️

Kernel

Isolating event bus, process table, module registry, and full syscall dispatch. The CPU SYSCALL instruction routes directly into OS actions — making AIOSCPU a first-class kernel citizen.

🗂

In-Memory Filesystem

POSIX-like Virtual File System with mkdir, ls, cat, write, mv, rm, stat, and tree. Files persist for the lifetime of the session.

🔀

Command Router

Hot-swap plug-in dispatcher. Modules register command handlers at boot and can be added or replaced at runtime without restarting the OS.

🛠

Service Manager

Start, stop, and restart named background services with svc. Services are isolated, lifecycle-managed processes that live inside the kernel's process table.

🤖

AI Core

Multi-backend AI routing with circuit breakers, persistent context, learning data, and proactive suggestion events. Wired to the Memory Core for unified cognitive output.

💡

AIOS + AURA

AIOS is the always-on AI personality (phone-first models). AURA is the on-demand hardware intelligence layer. Multi-turn /api/chat endpoint. Works fully offline.

🧬

Consciousness Layer

Memory Engine, Mode Manager, Model Registry, Diagnostics Engine, Port Server, and the unified Consciousness module — a cognitive runtime wired to the kernel bus.

💻

Interactive Terminal

A full readline REPL with ANSI colours, command history, and tab hints. Runs on any Node.js ≥ 14 — no native dependencies, no build step, no internet.

📱

Termux-Native

Designed from the ground up to boot on Android via Termux. No root, no compilation, no package manager needed beyond pkg install nodejs.

🔁

CPU–Kernel Self-Hosting Loop

Programs running on the virtual CPU use SYSCALL to reach the real OS. Read files, print output, trigger shutdown — all from within a CPU program.

🔒

Kernel Lockdown coming soon

The kernel is being hardened into a frozen, tamper-proof foundation. All new features will be built outside it as external modules against a read-only kernel proxy — keeping the core stable and auditable.

How it all fits together

A single ./aos entry point spins up the full OS in 27 ordered boot steps.

./aos └─ boot/bootstrap.js ← 27-step OS boot sequencer ├─ core/kernel.js Event bus · process table · syscall table ├─ core/filesystem.js In-memory VFS (POSIX-like) ├─ core/cpu.js AIOSCPU v1.0 (32-bit virtual processor) ├─ core/router.js Command router (hot-swap modules) ├─ core/service-manager.js Service lifecycle ├─ core/ai-core.js Multi-backend AI routing + circuit breakers ├─ core/memory-core.js Unified cognitive layer ├─ core/consciousness.js Consciousness layer orchestrator ├─ core/aios-aura.js AIOS personality + AURA hardware intelligence └─ terminal/terminal.js readline REPL (user shell) CPU–Kernel self-hosting loop: AIOSCPU program └─ SYSCALL instruction └─ kernel.syscall(num, args) └─ OS action (write / fs / exit / …) └─ kernel.bus.emit('kernel:syscall', …)

Built-in shell commands

Everything available at the interactive terminal once AIOS boots.

📁 Filesystem

  • pwdPrint working directory
  • cdChange directory
  • mkdirCreate directory (supports -p)
  • lsList directory contents
  • touchCreate empty file
  • catRead a file
  • writeWrite content to file
  • rmRemove file or directory (-r)
  • cpCopy file
  • mvMove / rename
  • statShow file metadata
  • treeRecursive directory listing

⚙️ OS / System

  • unameOS version string
  • uptimeSeconds since boot
  • psList running processes
  • killKill a process by PID
  • envShow environment variables
  • dateCurrent date/time (ISO 8601)
  • whoamiCurrent user
  • hostnameSystem hostname

🧠 AIOSCPU

  • cpu demoRun Hello World on the AIOSCPU
  • cpu infoShow CPU spec and state
  • cpu regsDump all register values
  • cpu resetReset the CPU

🛠 Service Manager

  • svc listList all services and states
  • svc statusDetailed service status
  • svc startStart a service
  • svc stopStop a service
  • svc restartRestart a service

🔀 Router / General

  • helpList all available commands
  • versionRouter version
  • echoEcho text back
  • historyCommand history
  • clearClear screen
  • exitShut down AIOS Lite

// quick start

Boot AIOS in 3 steps

No build tools, no npm install, no internet required.

📱 Android (Termux)

1

Install Node.js in Termux

pkg update && pkg install nodejs git
2

Clone the repository

git clone https://github.com/Cbetts1/AIOSCPU-PROTYPE.git
cd AIOSCPU-PROTYPE
3

Boot AIOS Lite

chmod +x aos
./aos

🖥 Desktop (Linux / macOS / Windows)

1

Clone the repository

git clone https://github.com/Cbetts1/AIOSCPU-PROTYPE.git
cd AIOSCPU-PROTYPE
2

Boot AIOS Lite

node aos
3

Start exploring

help          # list all commands
cpu demo      # run Hello World on AIOSCPU
svc list      # see running services
